Thread

A thread is a lightweight process and forms the basic unit of CPU utilization. A process can perform more than one task at the same time by including multiple threads.


  • A thread has its own program counter, register set, and stack.
  • A thread shares resources with other threads of the same process: the code section, the data section, files and signals

Note: -  A new thread, or a child process of a given process, can be introduced by using the fork() system call. A process with n fork() system call generates   2n–1 child processes.


Components of a Thread


Each thread has:

  • Program Counter
  • Registers
  • Stack

Threads share:

  • Code section
  • Data section
  • Open files/resources

Thread vs. Process


Feature Process Thread
Definition Independent executing program Lightweight unit of a process
Memory Has its own address space Shares memory with other threads
Overhead High (context switching is costly) Low (faster to create and switch)
Communication Complex (via IPC) Easy (shared memory)
Crash impact Doesn’t affect others Can crash the whole process

Types of Threads



  1. User-Level Threads (ULT):
    • Managed by user-level libraries.
    • OS doesn't know about them.
    • Fast creation and context switch.
    • But if one thread blocks, all do.


  2. Kernel-Level Threads (KLT):
    • Managed directly by the OS.
    • More overhead.
    • True parallelism on multi-core systems.


  3. Hybrid (Combined):
    • Combines both ULT and KLT (e.g., many-to-many model).
